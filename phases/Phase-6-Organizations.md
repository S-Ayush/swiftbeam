### 1.2 User Roles

| Role | Permissions |
|------|-------------|
| **Admin** | Create invites, remove members, delete org |
| **Member** | View members, see presence, share with members |

**Note:** Org creator automatically becomes Admin.

---

## 2. API Endpoints

### 2.1 Organization Routes

| Method | Endpoint | Purpose | Auth | Role |
|--------|----------|---------|------|------|
| POST | `/api/orgs` | Create org | Yes | - |
| GET | `/api/orgs` | List user's orgs | Yes | - |
| GET | `/api/orgs/:id` | Get org details | Yes | Member |
| DELETE | `/api/orgs/:id` | Delete org | Yes | Admin |
| PATCH | `/api/orgs/:id` | Update org name | Yes | Admin |

### 2.2 Member Routes

| Method | Endpoint | Purpose | Auth | Role |
|--------|----------|---------|------|------|
| GET | `/api/orgs/:id/members` | List members | Yes | Member |
| DELETE | `/api/orgs/:id/members/:userId` | Remove member | Yes | Admin |
| PATCH | `/api/orgs/:id/members/:userId` | Change role | Yes | Admin |

### 2.3 Invite Routes

| Method | Endpoint | Purpose | Auth | Role |
|--------|----------|---------|------|------|
| POST | `/api/orgs/:id/invites` | Create invite | Yes | Admin |
| GET | `/api/orgs/:id/invites` | List invites | Yes | Admin |
| DELETE | `/api/invites/:token` | Revoke invite | Yes | Admin |
| GET | `/api/invites/:token` | Validate invite | No | - |
| POST | `/api/invites/:token/accept` | Accept invite | Yes | - |

### 2.4 Request/Response Formats

**Create Org:**
```typescript
// Request
{ name: string }  // 3-50 chars

// Response
{
  id: string;
  name: string;
  createdAt: string;
  role: 'ADMIN';
}
```

**List Members Response:**
```typescript
{
  members: [{
    id: string;
    name: string;
    email: string;
    role: 'ADMIN' | 'MEMBER';
    joinedAt: string;
    isOnline: boolean;  // From presence system
  }];
}
```

**Create Invite Response:**
```typescript
{
  token: string;
  inviteUrl: string;  // Full URL
  expiresAt: string;
}
```
